Lesson · AI-Assisted Gamedev

AI × Phaser 遊戲上線

用 Claude Code、Codex、Gemini Antigravity 佈建 Phaser，把 HTML5 遊戲一路部署到 GitHub Pages。

你描述遊戲，AI 寫程式碼，
瀏覽器就是玩家的主機。

先認識引擎

Phaser 是什麼

免費、開源、快速的 2D HTML5 遊戲框架（Canvas + WebGL）。寫一次，桌機與手機瀏覽器都能玩，玩家開網址就能玩、零安裝。

為什麼選它

Phaser 的長處

  • 免費開源（MIT）— 商用不收授權費
  • 純網頁— 玩家開網址就能玩，零安裝，天生跨平台
  • JS / TypeScript— 跟 AI 編輯工具天生相搭
  • 生態龐大— 海量範例、文件、社群、外掛
  • 內建一切— 場景、物理、動畫、資源載入、輸入

2026/04 釋出 · 史上最大改版

Phaser 4 新在哪

  • 全新 WebGL 渲染器 — 更快、更穩、能優雅處理 context 遺失
  • 統一 Filter 系統 — Glow / Bloom / Vignette / Pixelate…
  • SpriteGPULayer — 百萬精靈一次繪製，最高快 100×
  • API 大致沿用 Phaser 3 — 舊知識與範例不浪費

本課用 Phaser 4；網路上大量 Phaser 3 教學與範例多數仍然適用。

Station 01

第一站
三個 AI 編輯工具

挑一個順手的就好

你的三個選擇

  • Claude Code（Anthropic）— 終端機 AI 代理，IDE 也能用npm i -g @anthropic-ai/claude-code　·　跑 Claude（Anthropic）
  • Codex CLI（OpenAI）— 終端機開源代理，Rust 寫成npm i -g @openai/codex　·　跑 GPT-5.5，sandbox 內執行
  • Gemini Antigravity（Google）— Agent 優先的 IDE App官網下載・免費預覽（非 npm）　·　跑 Gemini 3，編輯器 + Agent 管理台

三者用法相通

你是架構師，AI 是工人

  • 你用自然語言描述要什麼，AI 動手寫檔案、跑指令
  • 把目標講清楚、拆小步、每步做完先驗收再往下
  • 不會的指令直接叫 AI 跑；出錯就把錯誤訊息貼給它修

下面每一張「提示」三個工具都通用——差別只在介面，不在你要說的話。

Station 02

第二站
佈建 Phaser 環境

提示 ①　給 AI 的話

建立專案骨架

你 ▸ 用 Vite + Phaser 4 建立一個最小專案， 包含 index.html 與 src/main.js， 畫面顯示一個會左右移動的方塊。 ── AI 會幫你跑 ── npm create vite@latest my-game -- --template vanilla cd my-game && npm install phaser

Vite 是什麼？現代前端的開發工具：本地啟動快、存檔即時刷新，最後一鍵 build 成能部署的靜態檔。細節不用懂，AI 會處理。

看一眼產物

AI 生出的結構

my-game/ ├── index.html # 遊戲容器頁 ├── src/main.js # Phaser 設定 + 場景 ├── package.json # 依賴與指令 └── vite.config.js # 打包設定

看不懂結構？直接問 AI：「逐檔解釋這專案每個檔案在做什麼」。

提示 ②

本地跑起來

你 ▸ 幫我把開發伺服器跑起來 ── AI ── npm run dev # 開瀏覽器看 http://localhost:5173

看到方塊在動 = 環境 OK，可以開始做遊戲了。

Station 03

第三站
做出一個會玩的東西

提示 ③

一個最小可玩場景

你 ▸ 在這個 Phaser 場景加上： ・玩家方塊用方向鍵移動 ・從上方隨機掉落的方塊 ・碰到就 Game Over，按空白鍵重來 ── AI 直接改 src/main.js，存檔後瀏覽器自動刷新 ──

在瀏覽器玩玩看

驗收這一步

  • 方向鍵能控制玩家方塊
  • 看得到從上方掉落的方塊
  • 撞到會結束、能按鍵重來

哪裡怪 → 把畫面截圖或 console 錯誤貼給 AI，叫它修。這就是 AI 協作的循環。

Station 04

第四站
版本控制（接 Git）

這裡開始需要 Git

先有 Git，才能上線

  • 接下來要把遊戲推上 GitHub、再用 Pages 發布
  • 會用到 Git 安裝、設定、add / commit / push、Pages 設定
  • 不熟的話，先回去把基礎補起來再回來

👉 本站課程：Git / GitHub 入門（COURSE 01）

提示 ④

推上 GitHub

你 ▸ 幫我把這專案 git 初始化， 建一個公開 repo 叫 my-game 並推上去 ── AI ── git init && git add . && git commit -m "init: phaser game" gh repo create my-game --public --source=. --push

gh 沒登入？叫 AI 跑 gh auth login（細節見 Git 課第二、三站）。

Station 05

第五站
部署到 GitHub Pages

提示 ⑤

設子路徑 + build

你 ▸ 這要部署到 GitHub Pages 的 /my-game/ 子路徑， 幫我設好 Vite 的 base 並 build ── AI 改 vite.config.js ── export default { base: '/my-game/' } # 然後 npm run build # 產生 dist/

base 設錯 = 上線後一片空白、資源載不到——這是最常見的地雷。

提示 ⑥

把 dist/ 發布

你 ▸ 把 build 出來的 dist/ 發布到 gh-pages 分支 ── AI（手動部署）── cd dist && git init && git checkout -b gh-pages git add -A && git commit -m "deploy" git push -f <repo> gh-pages # 或：設 .github/workflows 自動部署，push 即上線

上線後檢查

驗收 & 地雷

  • 開 https://你的帳號.github.io/my-game/ 試玩
  • 一片空白 → base 路徑沒設對（回提示 ⑤）
  • 看到舊版 → 瀏覽器 / Service Worker 快取，強制重新整理
  • Settings → Pages 來源要選對分支（見 Git 課第五站）

出問題最快解法：把網址 + console 錯誤貼給 AI。

帶走什麼

你現在能做的

  • 用 AI 工具從零搭一個 Phaser 遊戲環境
  • 用自然語言「提示」讓 AI 寫遊戲、跑指令
  • 把 HTML5 遊戲免費部署到 GitHub Pages
